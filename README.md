# Legato — Private Payroll PoC

A minimal private payroll proof of concept for employers to set up payrolls, share claim links with employees, and for employees to privately claim their salaries without revealing which employee they are.

Built with [Noir](https://noir-lang.org) and deployed on Tempo's Moderato testnet.

> **Note:** The project sets out to be a minimal proof of concept, intentionally leaving advanced features out of scope (e.g. time-based unlocks, cross-employer privacy set, bigger company sizes).
>
> The project is not audited and is by no means production ready, do not deploy and use in production.

## Local development

### Run the app

Clone this repository:

```bash
git clone https://github.com/Savio-Sou/Legato.git
```

Run the app:

```bash
cd frontend
npm install
npm run dev
```

Open <https://localhost:3000> in your web browser to access the app.

### Rebuild the project

For if the circuits or contracts are changed and require redeployment.

#### 1. Compile circuits and generate Solidity verifier

```bash
cd circuits/payroll
nargo test            # run the circuit tests
nargo compile         # → target/payroll.json

# Generate the EVM verification key, then the Solidity verifier.
# Both steps MUST use the same `-t evm` target flag.
bb write_vk -t evm -b ./target/payroll.json -o ./target
bb write_solidity_verifier -t evm -k ./target/vk_evm -o ../../contracts/src/UltraVerifier.sol

# The frontend loads the compiled circuit from public/
cp ./target/payroll.json ../../frontend/public/payroll.json
```

#### 2. Deploy contracts

Build the smart contracts:

```bash
cd ../contracts
forge build
forge test
```

Create a _.env_ file:

```bash
cp .env.example .env
```

Generate or extract a deployer wallet's private key and copy it into the _.env_ file. Remember to fund it with testnet gas asset.

Deploy the smart contracts:

```bash
forge script script/Deploy.s.sol --rpc-url tempo --broadcast
```

#### 3. Update smart contract addresses

Edit the defaults in [frontend/src/lib/contracts.ts](frontend/src/lib/contracts.ts)

OR

Copy [frontend/.env.example](frontend/.env.example) to `frontend/.env.local` and set:

```bash
NEXT_PUBLIC_VERIFIER_ADDRESS=0x…
NEXT_PUBLIC_PAYROLL_MANAGER_ADDRESS=0x…
NEXT_PUBLIC_PATH_USD_ADDRESS=0x20C0000000000000000000000000000000000000
```

### End-to-end test

A Playwright harness drives the `/admin` and `/claim` UI against the live testnet and asserts on-chain results (payroll activated, contract funded, proof generated in-browser, `claim` mined, contract pays out exactly the salary):

```bash
cd frontend
npm run test:e2e        # = bash scripts/run-e2e.sh
```

It uses the dev connector (so it needs no passkey), reads the funder key from `contracts/.env`, and each run rotates a fresh employee address that it funds with pathUSD for gas. Requires the global `playwright` package and a Chromium that can launch. (Firefox/WebKit binaries are installed but need system libs to run; the dev connector works in them too once those are present.) Files live in `frontend/tests/e2e/` and `frontend/scripts/run-e2e.sh`.
